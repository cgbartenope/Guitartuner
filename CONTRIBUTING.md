# Contributing to Guitar Tuner Headquarters

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue on GitHub with:
- A clear, descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Browser and OS information

### Suggesting Enhancements

We welcome suggestions! Please create an issue with:
- A clear description of the enhancement
- Why this enhancement would be useful
- Any examples or mockups (if applicable)

### Pull Requests

1. **Fork the repository** and create your branch from `main`
2. **Make your changes** with clear, descriptive commit messages
3. **Test your changes** in multiple browsers
4. **Update documentation** if you're changing functionality
5. **Submit a pull request** with a clear description of your changes

### Code Style

- Use consistent indentation (2 or 4 spaces)
- Write semantic HTML
- Keep CSS organized and commented
- Use meaningful variable and class names
- Add comments for complex logic

### Adding New Content

When adding tuners, apps, or resources:
- Ensure all links are valid and working
- Include accurate product information
- Add proper descriptions and features
- Maintain consistent formatting

### Testing

Before submitting, please test on:
- Desktop browsers (Chrome, Firefox, Safari, Edge)
- Mobile devices (iOS and Android)
- Different screen sizes (responsive design)

## Questions?

Feel free to open an issue for questions or clarifications!
